<objective>
Track which safety defaults the user explicitly deselects so that switching away from a target and back does not re-add them. Currently, the target switch handler in prompt-guide.tsx (lines 298-304) unconditionally merges all safetyDefaults via `new Set([...currentConstraints, ...tool.safetyDefaults])`. If a user deselects `readable_text` on Seedance, switches to Generic Video, then back to Seedance — `readable_text` is silently re-added.

Purpose: Per D-01, "If user explicitly deselects a safety default, respect the deselection." The current implementation violates this contract on target re-switch. A `useRef<Set<string>>` tracking deselected safety default IDs provides lightweight, re-render-free tracking.

Output: Modified prompt-guide.tsx with deselection tracking, new component test in prompt-guide.test.tsx
</objective>

Replace with:
```typescript
  function updateSelection(questionId: string, value: SelectionValue) {
    // Track safety default deselections/re-selections for the constraints question
    if (questionId === "constraints" && Array.isArray(value)) {
      const currentTool = resolveTarget(targetToolId);
      const valueSet = new Set(value);
      for (const sdId of currentTool.safetyDefaults) {
        if (valueSet.has(sdId)) {
          deselectedSafetyRef.current.delete(sdId);
        } else {
          deselectedSafetyRef.current.add(sdId);
        }
      }
    }
    setSelections((current) => ({ ...current, [questionId]: value }));
  }
```

This uses `resolveTarget(targetToolId)` to get the current target's safetyDefaults. For each safety default ID: if it's in the new value, user re-selected it — remove from tracking. If it's not in the new value, user deselected it — add to tracking. The mutation happens before `setSelections` so the ref is always up-to-date for the next target switch.
