**Document structure and required sections:**

```
# Canva Brief — Forward Research for v2

## 1. What is a Canva Design Brief?
- Canva is a graphic design platform (social media posts, presentations, posters, branding kits)
- A "design brief" specifies what to create: format, dimensions, layout, typography, color palette, branding, content
- Fundamentally different from a video/cinematography prompt: no camera, no lighting, no motion, no audio
- Target user: someone who needs a specific design asset but doesn't know design terminology

## 2. Design Brief vs Video Prompt — Key Differences
| Aspect | Video Prompt | Design Brief |
|--------|-------------|--------------|
| Output | Time-based (duration, motion) | Static (single image/page) |
| Core dimensions | Shot type, camera movement, lighting, motion | Layout, typography, color palette, branding |
| Temporal elements | Duration, pacing, transitions | Not applicable |
| Format concerns | Aspect ratio, FPS | Dimensions (px), format (PNG/SVG/PDF), orientation |
| Text handling | On-screen text timing | Typography hierarchy (heading/body/caption) |
| Safety constraints | No celebrities, no IP | Brand compliance, copyright-safe assets |

## 3. New Dimensions Needed
Document each dimension with rationale:

### 3.1 Format / Dimensions
- Social media presets: Instagram post (1080x1080), Story (1080x1920), Facebook cover (820x312), Twitter header (1500x500)
- Document formats: A4, US Letter, Presentation (16:9, 4:3)
- Digital ads: Leaderboard (728x90), Skyscraper (160x600), Medium rectangle (300x250)
- Custom dimensions option

### 3.2 Layout Style
- Grid-based, asymmetric, single-hero, split-screen, magazine, minimalist, maximalist
- Hierarchy: headline-first vs visual-first
- White space: generous vs compact

### 3.3 Typography
- Font pairings (heading + body)
- Type scale (size hierarchy)
- Font categories: serif, sans-serif, display, script, monospace
- Alignment: left, center, right, justified
- Text density: minimal copy vs content-heavy

### 3.4 Color Palette
- Preset palettes (warm, cool, monochromatic, complementary, analogous, triadic)
- Brand color extraction (primary, secondary, accent)
- Contrast requirements (WCAG AA minimum)
- Background: solid, gradient, image-backed, transparent

### 3.5 Branding Elements
- Logo placement (top-left, centered, watermark)
- Brand patterns / textures
- Icon style (line, filled, duotone, custom)
- Photography style (lifestyle, product-on-white, abstract, illustrated)
- Illustration style (flat, 3D, hand-drawn, geometric)

### 3.6 Content / Messaging
- Headline length (short 3-5 words, medium 6-10, long 10+)
- Call-to-action type (Shop Now, Learn More, Sign Up, Download, Contact)
- Tone of voice (professional, playful, urgent, inspirational, minimal)
- Data visualization needs (charts, infographics, stats)

## 4. Dimensions No Longer Relevant (from video prompt)
These video-specific dimensions would be REMOVED for the design brief work type:
- Scene, Motion, Camera movement, Shot type, Lighting, Audio, Duration
- (These are cinematography concepts with no equivalent in graphic design)

## 5. Adapter Design Direction
- Follow the existing config-driven pattern: new `WorkTypeConfig` + `OptionSet` catalogs + `CanvaBriefRenderer`
- The renderer would produce a structured design brief in Markdown/natural language, NOT a Canva API call
- Output formats: Chinese brief, English brief, JSON spec, Markdown
- Registration: `registerWorkType(canvaBriefWorkType)` + `registerAdapter({ target: canvaBriefTarget, render: renderCanvaBrief })`
- No Canva API integration — the brief is copy-pasted by the user into Canva's design tools
- Option mapping: each design dimension becomes an `OptionSet` with consumer-friendly labels + professional design terms

## 6. Explicitly NOT Planned
Per project decisions (D-12):
- **No model-specific video adapters**: Sora, Runway Gen-4, Kling, Pika, Haiper — NOT planned. The Generic Video adapter already handles model-agnostic video prompts.
- **No additional video targets**: The adapter pattern stops at Seedance 2.0 + Veo 3 + Generic Video.
- **No Canva API integration**: The design brief is human-readable output, not an API payload.
- **No image generation**: This is about design specification, not AI image generation.

## 7. Implementation Estimate (v2)
- New work type config: 1 file (~80 lines)
- New option catalogs: 5-6 files (layout, typography, color, branding, content, format) — ~150 options total
- New renderer: 1 file (~120 lines)
- UI: reuse existing PromptGuide component (auto-discovers dimensions from work type config)
- Tests: ~20 new tests
- Estimated effort: 5-6 plans in a dedicated v2 phase

## 8. Open Questions for v2 Planning
- What's the actual Canva user workflow? (Do they start from a template or a blank canvas?)
- Should the brief include template recommendations? (Canva has 250K+ templates)
- Should color palette output include hex codes? (useful for brand consistency)
- Should typography recommendations reference Google Fonts? (Canva's font library overlaps significantly)
```
