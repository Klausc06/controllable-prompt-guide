**Step 1: Add `validateVersionConsistency()` to validation.ts**
Add a new exported function that checks all version fields are consistent within each catalog:
- Iterate over all option sets via `getAllOptionSets()`
- For each OptionSet, check that `set.version` matches every `option.version` within that set
- Return `string[]` of errors (empty = success) following the existing validation pattern
- Also cross-check against a target version constant if desired (e.g., all should be `"0.1.0"`)

```typescript
export function validateVersionConsistency(
  optionSets_?: OptionSet[],
  expectedVersion?: string
): string[] {
  const errors: string[] = [];
  for (const set of (optionSets_ ?? getAllOptionSets())) {
    for (const option of set.options) {
      if (option.version !== set.version) {
        errors.push(
          `Version mismatch in "${set.id}": option "${option.id}" has "${option.version}", ` +
          `expected "${set.version}" (set-level version)`
        );
      }
    }
    if (expectedVersion && set.version !== expectedVersion) {
      errors.push(
        `Option set "${set.id}" version "${set.version}" does not match expected "${expectedVersion}"`
      );
    }
  }
  return errors;
}
```
