```typescript
  defaultSelections: PromptSelections

    advancedOpen: false,
    deselectedSafety: new Set<string>()
  };
}
```

**Reducer implementation (four action paths):**

1. TARGET_CHANGED (per D-04 + RESEARCH.md Pitfall 1):
   - Import `resolveTarget` from registry, `getTargetsForOption` from registry
   - For each entry in state.selections: if value is string[], filter to options where `getTargetsForOption(optionId).includes(action.to)`. If filtered array is empty, remove the key entirely (silent discard). If value is string (free_text), keep it.
   - Resolve new target config via `resolveTarget(action.to)`.
   - Merge safetyDefaults: compute `safetyToAdd = newTarget.safetyDefaults.filter(id => !state.deselectedSafety.has(id))`. Union with current constraints selection. Use `new Set()` for dedup, NOT mutating previous state.
   - Return new state with `targetToolId: action.to` and filtered selections.

2. OPTION_SELECTED:
   - Get the question schema from the workType (resolveWorkType("video_prompt").questions). Find question where `q.id === action.questionId`. If not found, return state unchanged.
   - If mode is "single": set `selections[questionId] = action.optionId`.
   - If mode is "multi": get current array, add optionId if not present, enforce maxSelections cap via `.slice(0, question.maxSelections)`.
   - If mode is "free_text": this action should not fire for free_text — free_text uses OPTION_SELECTED with the text as optionId? No — free_text is handled directly. For safety, ignore OPTION_SELECTED/DESELECTED on free_text questions.
   - PER RESEARCH.md Pitfall 3: maxSelections enforcement MUST happen in reducer, not just in brief builder. If adding would exceed cap, rejection is silent.

3. OPTION_DESELECTED:
   - If mode is "single": remove the key from selections entirely.
   - If mode is "multi": remove optionId from the array. If array becomes empty, remove key from selections.
   - Safety tracking (per RESEARCH.md Pitfall 1): If the deselected option is a safety default of the current target AND `questionId === "constraints"`, add `action.optionId` to a new `deselectedSafety` Set. Create new Set: `new Set([...state.deselectedSafety, action.optionId])`.

4. TOGGLE_ADVANCED:
   - Return `{ ...state, advancedOpen: !state.advancedOpen }`.

IMPORTANT rules (RESEARCH.md anti-patterns):
- NEVER mutate previous state. Use `new Set()`, spread operators, `new Map()`.
- Never call throwing functions inside pure reducer paths without try/catch. Since `resolveTarget` throws on unknown ID, but TARGET_CHANGED only receives valid IDs from UI buttons, this is safe. If you want defense-in-depth, catch and return state unchanged.
- The reducer is a pure function — no registry calls that depend on runtime ordering beyond what's registered at import time.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI event → reducer action | User clicks dispatch action objects. Action type and payload come from component code, not user-controlled strings. |
| Reducer → registry | Reducer calls resolveTarget() and getTargetsForOption() which throw on unknown IDs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Tampering | useReducer action dispatch | mitigate | TypeScript discriminated union on `PromptGuideAction.type` — compiler enforces correct payload shape per action type. Exhaustive switch check catches missing cases at compile time. |
| T-03-04 | Spoofing | TARGET_CHANGED `from`/`to` fields | accept | Values come from registered target IDs rendered by getAllTargets(). User cannot inject arbitrary IDs. resolveTarget() throws on unknown ID as defense-in-depth. |
| T-03-05 | Elevation of Privilege | OPTION_SELECTED optionId | accept | optionId comes from registered options; getOptionById returns undefined for unknown IDs. Brief builder filters out undefined. |
</threat_model>
